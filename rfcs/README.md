# Namel3ss RFCs (Requests for Comments)

This directory contains RFCs (Requests for Comments) for proposed changes to the Namel3ss language specification.

## Active RFCs

Currently no active RFCs. Check the [RFC tracker](https://github.com/your-org/namel3ss/labels/rfc) for proposals in progress.

## RFC Process

See [GOVERNANCE.md](../GOVERNANCE.md) for the full RFC process.

### Quick Start

1. Copy `0000-template.md` to `NNNN-my-feature.md` (use next available number)
2. Fill out the template
3. Submit as a pull request with title "RFC NNNN: [Title]"
4. Announce in GitHub Discussions
5. Iterate based on feedback
6. Maintainers will guide through FCP and decision

### RFC Lifecycle

- **Draft** → **Discussion** → **Final Comment Period** → **Accepted/Rejected** → **Implemented**

### What Needs an RFC?

- New language syntax or features
- Breaking changes to existing features
- Changes to language semantics
- Major standard library additions
- Deprecations

### What Doesn't Need an RFC?

- Bug fixes
- Performance improvements
- Documentation updates
- Implementation-only changes (CLI flags, internal refactoring, etc.)

## Implemented RFCs

Once implemented, RFCs are moved to `implemented/` for historical reference.

## Rejected RFCs

Rejected RFCs are moved to `rejected/` with rationale preserved.
